# 项目优化记录

## 2026-02-11

### 1. Message格式改为拆分格式
**文件**: `src/core/builder.py`
**优化点**: 将原来单一的JSON结构拆分为多个独立消息，使Message List结构更清晰

**Message List组成**:
- `system` - 系统提示词
- `user` - 用户原始输入
- `user` - 重写后的意图
- `user` - 每个工具的决策结果（分开显示）
- `user` - 最近15轮对话历史
- `user` - 最终回答请求

### 2. 增加详细终端日志输出
**文件**: `src/core/agent.py`
**优化点**: 每一轮对话显式输出完整的处理流程和数据流向

**日志输出阶段**:
- **[Stage 1]** ReQuery阶段：显示原始输入、重写后意图、提取参数、ReQuery耗时
- **[Stage 2]** 工具智能体决策：显示每个工具的USE_TOOL=True/False、原因、结果预览
- **[Stage 3]** Message List构建：显示携带的历史消息数量、每条消息的role和内容预览
- **[Stage 4]** 主模型生成回复
- **[Stage 5]** TTS语音合成
- **[Stage 6]** 更新对话历史

### 3. 添加首段音频播放时间记录
**文件**: `src/core/agent.py`, `src/utils/performance.py`
**优化点**: 新增首段音频播放时间(TTFA)监控

**修改内容**:
- `TTSTask`类新增：
  - `_first_audio_play_time`字段
  - `_generation_start`字段
  - `set_generation_start()`方法
  - `get_first_audio_time()`方法
- `PerformanceMetrics`类新增：
  - `first_audio_time`字段
  - `set_first_audio_time()`方法
- `print_report()`方法新增TTFA输出

**性能指标**:
- `TTFT`(Time To First Token) - 首token延迟
- `TTFA`(Time To First Audio) - 首段音频延迟

### 4. 修复性能计算方法
**文件**: `src/core/agent.py`
**优化点**: 修复`PerformanceMetrics`缺少`get_duration_ms`方法的问题，改为直接访问`stage_times`字典计算耗时

### 5. TTS触发策略优化 - 首字符优先+标点断句
**文件**: `src/core/agent.py`
**优化日期**: 2026-02-11
**优化目标**: 减少首音频延迟(TTFA)，优化音频切分符合人类交流习惯

**修改内容** - `TTSTask._should_trigger()`:
- **首字符立即触发**：buffer达到1个字符时立即触发第一段TTS
- **标点断句**：只在真正断句的标点符号处切分（。！？）
- **跳过特殊符号**：不因省略号、破折号等特殊符号触发切分

**效果**:
- 首音频延迟(TTFA)预计从2500ms降低到500-800ms
- 音频切分更符合自然语言习惯
- 减少不必要的碎片化

### 6. TTS同步模式优化
**文件**: `src/services/tts.py`
**优化日期**: 2026-02-11
**优化目标**: 确保TTS稳定可用

**修改内容**:
- 移除不稳定的`synthesize_stream()`流式方法
- 使用同步`synthesize()`方法确保稳定性
- 配合前端TTSTask的快速触发策略实现低延迟

**注意**: 真正的WebSocket流式TTS需要使用`dashscope-realtime` SDK，当前方案使用快速触发的同步TTS作为替代
