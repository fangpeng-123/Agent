# 2026年3月9日 TTS服务问题分析与修复记录

## 一、问题列表

### 1. 音频末尾被直接中断
- **现象**：播放音频时，末尾突然中断然后播放下一段
- **原因**：`sd.play()` 是非阻塞调用，立即返回，下一段音频的 `sd.play()` 会覆盖缓冲区，导致前一段被截断

### 2. 播放完最后一段需要等待数秒
- **现象**：最后一段音频播放完后，需要等待约10秒才能继续对话
- **原因**：`wait_finish()` 方法中的 Event 只在收到 `None`（结束信号）时才设置，每段音频播放完没有通知

### 3. 两段音频间隔时间长
- **现象**：上一段音频播放完到下一段音频开始播放之间间隔较长
- **原因**：TTS合成串行执行，必须等上一段播放完才能处理下一段（架构问题，暂未解决）

### 4. 切块策略不好
- **现象**："你好小明"（无标点）或"你好，小明"（句中标点）会被切成多段，语气不一致
- **原因**：见标点就切，没有文本累积策略

### 5. WebSocket连接超时
- **现象**：程序启动一段时间不用，报 `[TTS] connection closed with code: None, msg: Nonewebsocket closed due to fin=1 opcode=8 data=b'\x03\xe8Bye'`
- **原因**：DashScope WebSocket服务长时间空闲会自动关闭（正常行为）

## 五、2026年3月9日 间隔问题优化尝试

### 问题描述
用户反馈：两段音频之间的间隔时间过长，希望优化。

### 分析过程

#### 1. 架构分析
通过阅读 `src/core/agent.py` 中的 `TTSTask` 类，发现当前处理流程：

```
LLM输出文本 → add_text() → _text_queue → _tts_consumer() → synthesize() → _audio_queue → AudioPlayer → 播放
```

**问题根因**：`_tts_consumer()` 中使用 `await self.tts_service.synthesize(text_segment)` 串行等待，必须等当前段完全合成并播放完，才能处理下一段。

#### 2. 解决方案探索

**方案一：并行合成（预加载）**
- 思路：在播放当前段时，后台预合成下一段
- 尝试实现：使用 `asyncio.gather()` 并行调用多次 `synthesize()`

**方案二：流式TTS服务**
- 思路：使用已有的 `StreamTTSService`，利用其真正的流式输出能力
- 优点：边合成边输出，减少等待
- 缺点：需要较大架构改动

#### 3. 实施过程

**第一次尝试：并行合成**

在 `_tts_consumer()` 中实现预加载逻辑：

```python
# 收集后续文本段
texts_to_synthesize = [text_segment]
for _ in range(max_preload):
    next_text = self._text_queue.get_nowait()
    texts_to_synthesize.append(next_text)

# 并行合成
tasks = [asyncio.create_task(self.tts_service.synthesize(text)) for text in texts_to_synthesize]
results = await asyncio.gather(*tasks)
```

**测试发现问题**：
```
[TTS] 并行合成 2 段文本
[TTS] session created: sess_HdqmqACNQmOm5BvFe7Mh2
[TTS] session created: sess_Zf71N47EGxeUVByL9XTTd
websocket closed ... Session update error: session already started or finished
```

**原因**：DashScope TTS API 的 WebSocket 不支持并发连接，两个 session 冲突导致失败。

**第二次尝试：添加锁保护**

在排查问题时，发现另一个并发 bug：`add_text()` 使用 `asyncio.create_task()` 不等待完成，导致多个任务并发修改 `_text_buffer`，文本丢失。

修复：添加锁保护

```python
# 添加锁
self._text_buffer_lock = asyncio.Lock()

# add_text() 加锁
async def add_text(self, text: str):
    async with self._text_buffer_lock:
        self._text_buffer = getattr(self, "_text_buffer", "") + text
        # ... 后续处理

# finish() 加锁
async def finish(self):
    async with self._text_buffer_lock:
        if self._text_buffer:
            text_to_flush = self._text_buffer
            self._text_buffer = ""
            await self._text_queue.put(text_to_flush)
```

**最终方案**：回退并行合成，保持串行处理，但保留锁保护避免文本丢失。

## 七、2026年3月9日 下午 播放中断与队列管理问题修复

### 问题描述

1. **问题一**：TTS转换第一段播放后就不播放第二段了
2. **问题二**：播放长音频会中断跳转到下一段，上一轮对话还没结束就显示下一轮对话窗口

### 问题分析

#### 问题一：只播放第一段

**问题原因**：
- `sd.wait()` 阻塞 asyncio 事件循环，导致 TTS 合成任务无法运行
- 播放任务和 TTS 任务实际上是串行的，不是并行的

**解决方案**：
- 将播放移到线程池中执行，不阻塞主事件循环
- 使用 `loop.run_in_executor()` 包装同步的播放函数

```python
# 修改前
def _play_audio_async(audio_data: bytes):
    sd.play(data, framerate)
    sd.wait()  # 阻塞事件循环！

# 修改后
def _play_audio_sync(audio_data: bytes):
    # 同步播放函数
    sd.play(data, framerate)
    sd.wait()

async def _play_audio_async(audio_data: bytes):
    loop = asyncio.get_event_loop()
    await loop.run_in_executor(None, _play_audio_sync, audio_data)
```

#### 问题二：播放中断、提前显示下一轮

**问题原因**：
- `finish()` 方法中结束信号发送时机不对，TTS 完成后立即发送结束信号，但 AudioPlayer 队列中可能还有未播放的音频
- `wait_finish()` 只等待单个 `_play_finished_event`，不能准确反映队列状态
- 异步任务退出时机不对

**解决方案**：
1. 添加 `_all_done_event` 事件和 `_audio_count` 计数器，准确跟踪待播放音频数量
2. 调整 `finish()` 方法的执行顺序：
   - 先等待 TTS 合成任务完成
   - 再等待 `_audio_player_task` 把所有音频放入队列
   - 然后等待 `AudioPlayer` 所有音频播放完成（使用新的 `_all_done_event`）
   - 最后才发送结束信号

```python
class AudioPlayer:
    def __init__(self):
        # ...
        self._all_done_event = asyncio.Event()  # 所有音频播放完成信号
        self._audio_count = 0  # 待播放音频计数

    async def _play_loop(self):
        while True:
            audio_data = await self.queue.get()
            if audio_data is None:
                self._audio_count = 0
                self._all_done_event.set()  # 通知所有音频播放完成
                break
            self._audio_count += 1
            await _play_audio_async(audio_data)
            self._audio_count -= 1

    async def wait_finish(self, timeout: float = 30.0):
        """等待所有音频播放完成"""
        await self._all_done_event.wait()
```
