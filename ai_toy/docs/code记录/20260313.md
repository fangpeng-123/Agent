# 2026年3月13日 TTS 相关代码修改记录

## 修改概述

本次修改主要针对 TTS 服务的性能和用户体验进行了优化，包括：

1. **WebSocket 连接复用** - 减少段间停顿和首段延迟
2. **TTFA 计时修正** - 准确计算首段音频延迟
3. **音频任务管理** - 解决旧音频残留问题
4. **淡出效果** - 防止音频段间爆音

## 修改详情

### 1. WebSocket 连接复用 (`src/services/tts/providers/qwen_tts.py`)

**问题**：每次调用都重新建立 WebSocket 连接，导致首段延迟和段间停顿明显。

**解决方案**：
- 添加 `_ensure_connection()` 方法，维护长连接
- 使用双检查锁定 (DCL) 保证线程安全
- 复用同一个 `QwenTtsRealtime` 实例发送多个文本段

```python
# 核心改动
self._tts_instance: Any = None
self._callback: Any = None
self._connection_lock = threading.Lock()

def _ensure_connection(self):
    if self._tts_instance is None:
        with self._connection_lock:
            if self._tts_instance is None:
                # 只在首次建立连接
                self._callback = StreamingCallback()
                self._tts_instance = QwenTtsRealtime(...)
```

**预期效果**：
- 首段延迟：从 ~500ms+ 降至 ~50ms
- 段间停顿：从 ~300ms+ 降至 ~0ms

### 2. 文本分割阈值优化 (`src/services/tts/base.py`)

**问题**：首段触发门槛过高。

**解决方案**：
- `MIN_SEGMENT_LENGTH`: 8 → 4 (短句更快输出)
- `FAST_RESPONSE_LENGTH`: 15 → 8 (逗号分割更快响应)
- `LONG_SEGMENT_LENGTH`: 20 → 15 (长句更快拆分)

---

### 3. TTFA 计时修正 (`src/core/agent.py.bak`)

**问题**：TTFA (首段音频延迟) 计算不准确，漏掉了消息构建和 LLM 启动时间。

**解决方案**：
- 在 `_process_new()` 开始时记录 `request_start_time`
- 将 `request_start_time` 传给 `_generate_response()`
- 使用 `request_start_time` 作为 TTFA 计时起点
- TTFT (首 token 延迟) 单独计算，只包含 LLM 推理时间

```python
# 修改前
generation_start = time.time()  # LLM 开始生成时才计时

# 修改后
request_start_time = time.time()  # 用户请求开始时计时
generation_start = request_start_time  # TTFA 从用户请求开始
llm_start_time = time.time()  # TTFT 从 LLM 调用开始
```

**计时逻辑**：
| 指标 | 计时起点 | 计时终点 |
|------|----------|----------|
| TTFA | 用户请求开始 | 首段音频播放 |
| TTFT | LLM 调用开始 | 第一个 token |

### 4. 音频任务管理 (`src/core/agent.py.bak`)

**问题**：用户输入新消息时，之前的音频没有停止，继续播放上次残留的音频。

**解决方案**：
- 添加 `_prev_tts_task` 属性保存上一个 TTS 任务引用
- 添加 `_cancel_previous_tts_task()` 方法，在新请求开始时：

  - 清空 `_audio_queue` 队列

  - 取消旧的任务
- 在 `process()` 开始时调用取消方法

### 5. 音频淡出效果 (`src/core/agent.py.bak`)

**问题**：音频段与段之间播放时，衔接处可能出现爆音。

**解决方案**：
- 添加 `_apply_fade_out()` 函数
- 在 `_play_audio_sync()` 中播放前应用淡出效果
- 淡出时长：20ms (480 采样点 @24kHz)

```python
def _apply_fade_out(data: np.ndarray, fade_samples: int = 480) -> np.ndarray:
    fade_len = min(fade_samples, len(data))
    fade_curve = np.linspace(1.0, 0.0, fade_len)
    data[-fade_len:] = (data[-fade_len:] * fade_curve).astype(data.dtype)
    return data
```
