# 儿童教育陪伴系统开发记录

## 日期：2026-02-25

## 项目概述

基于 qwen3-235b-a22b 模型（训练时具备国学成语故事、格林童话等数据集）设计3-12岁儿童教育陪伴产品，涵盖国学、世界名著、寓言故事、历史人物、地理知识、自然科学、社会科学等领域。

## 需求确认过程

### 1. 知识库定位
**问题**：是否需要配备知识库或联网搜索功能？
**回答**：即使不配备知识库或联网搜索也可以依据事实回答问题。

### 2. 提示词设计
**需求**：
- 设计一套提示词让模型以更贴近儿童教育的口吻分享故事
- 剔除敏感字眼
- 根据用户信息（年龄、兴趣爱好、居住地）优化回复方向

**确认细节**：
- **3-6岁儿童**：注重故事分享，使用儿童化语言表达（Few-Shot Examples）
- **7-12岁儿童**：偏重故事寓意、知识讲解

### 3. 语言润色规则
**需求**：
- 修改成人化表达（如"他非常难过"改为"他的眼泪吧嗒吧嗒掉下来"）
- 纠正语法错误（如"我吃了苹果和香蕉"改为"我吃了苹果和香蕉，真甜呀"）
- 调整语言节奏（低幼读物需用"重复+变化"的句式，如"小鸭子游呀游，游到小河里；小鸭子游呀游，游到荷叶边"）

**确认细节**：
- 6岁算3-6岁（学前儿童）类
- 语言重写只在系统提示词中约束模型输出时使用儿童化表达
- 3-6岁具体语言特征按给定用例做Few-Shot写在系统提示词中

### 4. 事实准确性
**问题**：是否需要设计校验机制？
**回答**：先不用开发，优先完善其他功能。

### 5. 交互形式
**确认**：端到端的语音交互，TTFT控制好就不会有问题。

### 6. 居住地信息使用
**确认**：只作为用户画像的一部分，只在查看天气或其他涉及的问题才调用。

### 7. 年龄分界点
**确认**：6岁算3-6岁类（学前儿童）。

### 8. 知识领域优先级
**确认**：
- 优先讲故事，然后介绍人物
- 3-6岁：只讲故事
- 7-12岁：添加寓意讲解
- 超出训练范围：使用委婉语气告知暂时不知道

## 2026-02-25 下午问题修复记录

### 问题1：Unicode 代理字符编码错误

**错误信息**：
```
UnicodeEncodeError: 'utf-8' codec can't encode character '\udcb7' in position 1649: surrogates not allowed
```

**原因**：LLM 返回或输入中包含无效的 Unicode 代理字符 `\udcb7`，该字符无法用 UTF-8 编码

**影响文件**：`src/core/requery.py`

**解决方案**：
1. 添加 `_clean_string()` 函数清理字符串中的无效代理字符
2. 在构建 prompt 和 messages 时调用该函数

```python
def _clean_string(s: str) -> str:
    """移除无效的 Unicode 代理字符"""
    try:
        return s.encode("utf-8", "surrogatepass").decode("utf-8", errors="replace")
    except Exception:
        return s
```
