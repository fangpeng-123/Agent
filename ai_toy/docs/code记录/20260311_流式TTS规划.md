# 2026年3月11日 阿里云流式TTS服务接入规划

## 一、问题描述

### 当前问题

| 问题 | 现象 | 原因 |
|------|------|------|
| 首句等待久 | 用户输入后需要等待较长时间才能听到第一句话 | 文本累积到60字才触发TTS合成 |
| 播放不连贯 | 播放过程中有明显的停顿或延迟 | 累积式流式：一段一段地合成播放 |
| 语义截断 | 硬编码切分导致句子从中间被截断 | MAX_CHUNK_LENGTH=60 可能截断句子 |

### 用户期望

- 不改变硬编码策略（保持句子完整性）
- 保持当前整句/段落的流式粒度
- 减少首句延迟
- 实现更流畅的播放体验

## 二、阿里云TTS服务分析

### 2.1 可用的TTS服务

| 服务类型 | 模型 | 特点 |
|----------|------|------|
| 语音合成-千问（批量） | qwen3-tts-flash | 非流式，适合长文本 |
| **实时语音合成-千问** | qwen3-tts-instruct-flash-realtime | **流式输出，低延迟** |
| CosyVoice实时合成 | cosyvoice-v3-plus | 支持音色克隆，高质量 |

### 2.2 实时语音合成核心特性

根据阿里云官方文档：

| 特性 | 说明 |
|------|------|
| **流式输出** | 支持流式文本输入与流式音频输出，边合成边播放 |
| **低延迟** | 端到端延迟 < 100ms，适合实时交互场景 |
| **多音色** | 支持 Cherry、Serena、Ethan、Chelsie 等 |
| **指令控制** | 可通过自然语言控制语音表现力 |
| **多语言** | 支持中英等多语种混合输入 |

### 2.3 接入方式

```python
# 阿里云 DashScope SDK
from dashscope import MultiModalConversation

response = MultiModalConversation.call(
    model="qwen3-tts-instruct-flash-realtime",
    text="需要合成的文本",
    voice="Cherry"
)
```

## 三、优化方案

### 3.1 当前架构（累积式流式）

```
LLM输出 → 文本缓冲区(累积到60字) → TTS合成(整段) → AudioPlayer播放
                                           ↑
                                    等待合成完成
```

**问题**：首段需要等待累积60字 + TTS合成完成

### 3.2 目标架构（真流式）

```
LLM输出 → 流式TTS(阿里云实时) → 边合成 → 边播放
                              ↑
                        低延迟流式输出
```

**优势**：
- 首字延迟 < 100ms
- 无需等待整段合成
- 播放更流畅

### 3.3 实现方案

新建流式TTS模块：

```
src/
└── services/
    └── tts/
        ├── base.py           # 基础TTS接口
        ├── streaming_tts.py  # 阿里云流式TTS（新增）
        └── qwen_tts.py       # 当前QwenTTS（保留兼容）
```
