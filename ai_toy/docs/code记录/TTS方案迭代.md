# TTS 方案迭代

## 2025-03-12 讨论记录

### 背景

分析旧版 TTS 是否使用两级切块策略，以及为什么这样做。

---

### 一、现有 TTS 架构（两级切块）

```
┌─────────────────────────────────────────────────────────────┐
│ 第一级：文本缓冲 (Buffer)                                     │
│ 来源：LLM 流式输出 / 用户输入文本                             │
│ 机制：tts_text_buff 累积文本，直到满足分割条件               │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 第二级：智能文本分割 (Segmentation)                           │
│ 核心方法：TTSProviderBase._get_segment_text()                │
│ 策略：根据标点类型 + 文本长度 动态决定切分点                  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 第三级：TTS API 调用                                          │
│ 每个 segment 调用一次 _process_tts_segment()                 │
│ 流式返回音频块                                                │
└─────────────────────────────────────────────────────────────┘
```

#### 现有分割规则

| 条件 | 行为 |
|------|------|
| 文本 ≥30 字 + 句末标点 | 整句保护，不切分 |
| 新文本块 >30 字 | 长文本保护，不切分 |
| 当前片段 <8 字 | 短句合并，继续累积 |
| 当前片段 ≥20 字 | 长句拆分 |
| 遇到句末标点(。？！；) | 立即分割 |
| 遇到逗号 + ≥8 字 | 分割触发 |

#### 现有方案延迟

- 最小长度阈值：8 字
- 首音延迟：约 300-500ms
- 最坏情况：等 LLM 输出 8 字（约 1-2 秒）

---

### 二、新方案：1-4 字合并策略

#### 核心规则

**标点符号前只有 1-4 个字就和下一段合并成一个块**

#### 示例

| 原始文本 | 分割结果 | 说明 |
|----------|----------|------|
| 你好，小明 | 你好，小明 | "你"只有1字，合并 |
| 今天天气，真不错 | 今天天气，真不错 | "天气"只有3字，合并 |
| 今天天气真不错，我们去玩吧 | 今天天气真不错，我们 | "天气"后是逗号但前面有5字，不合并 |
| 啊，原来是這樣 | 啊，原来是這樣 | "啊"只有1字，合并 |

#### 长度控制

- 通过 System Prompt 让 LLM 输出短句
- 不输出长句

---

### 三、两种方案对比

| 维度 | 现有方案（8字+逗号） | 新方案（1-4字合并） |
|------|---------------------|-------------------|
| 触发条件 | 累积≥8字 或 遇到句末标点 | 按标点切分 + 1-4字合并 |
| LLM 输出要求 | 无特殊要求 | 要求输出短句 |
| 首音延迟 | 取决于 LLM 输出 8 字的速度 | 取决于 LLM 输出短句的速度 |
| 最坏情况 | 等 8 字（约 1-2 秒） | 等一个短句（约 0.5-1 秒） |
| 语音碎片度 | 较高（8字起） | 较低（短句更自然） |
| 实现复杂度 | 已实现 | 需重新设计 |

---

### 四、方案评估结论

**新方案理论上延迟更低**

原因：
1. LLM 被要求输出短句，单次输出量更少
2. 1-4 字合并规则让触发更频繁
3. 不需要等累积到 8 字
