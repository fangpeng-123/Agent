在 LangChain 和 LangGraph 中调用模型的方式多样，核心目标是让开发者能灵活、高效地集成各种大语言模型（LLM）到应用中。下面我将这些方法梳理成一个表格，方便你快速了解，然后再提供一些示例和总结。

### 📊 模型调用方式概览

| 调用方式                     | 适用场景                                                     | 关键工具/库                               | 主要特点                                              |
| :--------------------------- | :----------------------------------------------------------- | :---------------------------------------- | :---------------------------------------------------- |
| **1. 原生API直接调用**       | 快速测试、简单集成                                           | `requests`, `openai` 库                   | 直接控制，灵活度高，但需手动处理上下文、格式化等逻辑  |
| **2. LangChain 框架调用**    | 构建复杂应用链（Chain），需连接多个组件（模型、工具、数据源） | `langchain.chat_models`, `langchain.llms` | 组件化设计，集成度高，支持多种模型提供商和本地模型    |
| **3. LangGraph 有状态代理**  | 构建具备记忆、循环、工具调用等能力的多步骤智能体（Agent）    | `langgraph.graph`, `langgraph.prebuilt`   | 支持复杂工作流和状态管理，适合多轮对话和决策流程      |
| **4. Ollama 本地模型调用**   | 在本地或私有环境中运行和调用开源模型                         | `ollama` Python 库, `ollama run` 命令     | 本地运行，数据隐私性好，支持多种开源模型              |
| **5. SGLang 高性能服务框架** | 高性能推理、流式输出、复杂采样控制，需部署自有模型           | `sglang`                                  | 为LLM/VLM设计的高效服务框架，提升推理速度和资源利用率 |
| **6. 第三方平台API调用**     | 快速使用特定厂商（如DeepSeek, Tongyi, Zhipu）的强大模型，无需管理基础设施 | 各平台提供的SDK或API Key                  | 免部署，快速上手，依赖网络和API配额                   |

### 🚀 各类调用方法详解与示例

#### 1. LangChain 基础模型调用

LangChain 通过统一的接口封装了多种模型的调用细节，无论是云端API还是本地模型。

```python
from langchain.chat_models import ChatOpenAI
from langchain.schema import HumanMessage, SystemMessage

# 1. 调用国内深度求索等遵循 OpenAI 规范的模型 :cite[1]
chat = ChatOpenAI(
    model="Qwen/Qwen2.5-7B-Instruct",
    openai_api_key="your_api_key",  # 替换为你的 API Key
    openai_api_base="https://api.deepseek.com",  # 或对应的 API Base URL
    temperature=0.7,
    max_tokens=500
)

# 2. 构建消息并调用
messages = [
    SystemMessage(content="你是一个乐于助人的助手。"),
    HumanMessage(content="请用 Python 写一个快速排序函数。")
]
response = chat.invoke(messages)
print(response.content)

# 对于 Hugging Face 或 ModelScope 的本地模型，可使用 HuggingFacePipeline
from langchain.llms import HuggingFacePipeline
import transformers

pipe = transformers.pipeline("text-generation", model="your-local-model-path", device=0)
llm = HuggingFacePipeline(pipeline=pipe)
response = llm.invoke("请介绍一下你自己。")
print(response)
```



#### 2. LangGraph 智能体与状态管理

LangGraph 允许你构建有状态的图（Graph），非常适合创建能进行多轮交互、调用工具的智能体（Agent）。

```python
from langgraph.graph import StateGraph, START, END
from langgraph.prebuilt import create_react_agent
from langchain_community.chat_models.tongyi import ChatTongyi  # 以通义千问为例 :cite[2]
from langgraph.checkpoint.memory import InMemorySaver  # 简单的内存记忆存储

# 初始化模型
llm = ChatTongyi(model="qwen-turbo", temperature=0)

# 定义一个简单的工具函数
def get_weather(city: str) -> str:
    """获取指定城市的天气信息"""
    return f"{city}的天气是晴天，25摄氏度。"

# 创建检查点存储以实现短期记忆 :cite[2]
checkpointer = InMemorySaver()

# 构建智能体
agent = create_react_agent(llm, tools=[get_weather], checkpointer=checkpointer)

# 调用智能体，使用相同的 thread_id 来维持对话上下文 :cite[2]
config = {"configurable": {"thread_id": "123"}}  # 会话唯一标识

# 第一轮提问
response1 = agent.invoke(
    {"messages": [{"role": "user", "content": "北京今天天气怎么样？"}]},
    config=config
)
print(response1["messages"][-1]["content"])

# 第二轮提问，智能体会记住之前的对话
response2 = agent.invoke(
    {"messages": [{"role": "user", "content": "我刚才问的是哪个城市？"}]},
    config=config
)
print(response2["messages"][-1]["content"])  # 应能回答“北京”
```



#### 3. Ollama 本地模型调用

Ollama 让你能方便地在本地运行各种开源大模型。

- **命令行直接调用** 

  ```bash
  # 拉取模型（例如 deepseek-r1:7b）
  ollama pull deepseek-r1:7b
  # 运行模型并进行问答
  ollama run deepseek-r1:7b "用 Python 实现一个简单的 HTTP 服务器"
  ```

  

- **在 Python 代码中通过 API 调用** 

  ```python
  import requests
  import json
  
  response = requests.post(
      "http://localhost:11434/api/generate",
      json={
          "model": "deepseek-r1:7b",
          "prompt": "为什么太阳会发光？",
          "stream": False
      }
  )
  result = response.json()
  print(result["response"])
  ```

  

- **使用 LangChain 集成 Ollama** 

  ```python
  from langchain_community.llms import OllamaLLM
  
  llm = OllamaLLM(model="deepseek-r1:7b")
  response = llm.invoke("解释一下神经网络的基本原理。")
  print(response)
  ```

  

#### 4. SGLang 高性能服务与函数调用

SGLang 是一个为高效服务大型语言模型和视觉语言模型设计的框架，特别优化了推理速度和资源利用率 。它支持复杂的采样控制和函数调用。

- **部署模型服务** (以 Qwen3-14B 为例) 

  ```bash
  # 通过 Docker 启动 SGLang 服务器
  docker run -d --gpus all \
    --name sglang-qwen3-14b \
    -p 8001:80 \
    -v /path/to/your/model:/models \
    swr.cn-north-4.myhuaweicloud.com/.../sglang:v0.4.7.post1-cu124 \
    python3 -m sglang.launch_server \
    --model-path /models/Qwen/Qwen3-14B \
    --served-model-name Qwen3-14B \
    --host 0.0.0.0 \
    --port 80
  ```

  

- **通过兼容 OpenAI 的 API 调用** 

  ```python
  import openai
  
  client = openai.OpenAI(
      base_url="http://localhost:8001/v1",  # SGLang 服务地址
      api_key="none"  # 若未设置认证，可使用任意字符串
  )
  
  response = client.chat.completions.create(
      model="Qwen3-14B",
      messages=[{"role": "user", "content": "写一首关于春天的诗。"}]
  )
  print(response.choices[0].message.content)
  ```

  

- **SGLang 的函数调用** 
  SGLang 支持让模型根据用户查询生成结构化输出来调用外部工具或 API，这为模型解锁了实时数据获取、数据库查询等能力 。

许多平台提供现成的大模型 API，你只需要注册获取 API Key 即可集成。

### 💎 总结与选择建议

选择哪种方式调用模型，主要看你的具体需求：

| 需求场景                               | 推荐方式                                  | 原因                                                         |
| :------------------------------------- | :---------------------------------------- | :----------------------------------------------------------- |
| **快速原型验证、简单应用**             | **LangChain 基础调用**、**第三方平台API** | 上手快，代码简洁，第三方API无需管理基础设施                  |
| **复杂多步应用、具备记忆的智能体**     | **LangGraph**                             | 内置状态管理和检查点机制，轻松处理多轮对话和工具调用         |
| **本地运行、数据隐私要求高、离线环境** | **Ollama**                                | 部署简单，支持大量开源模型，保证数据不出域                   |
| **高性能推理、需要流式输出、复杂控制** | **SGLang**                                | 框架针对推理速度和服务吞吐量进行了优化，支持复杂的采样控制和函数调用 |
| **集成特定厂商模型、利用其独特能力**   | **相应平台的 SDK 或 API**                 | 可以直接利用厂商提供的强                                     |





### 📋 核心方法一览：从本地到云端

| 调用方式                            | 核心思路                                                     | 适用场景                                                   | 关键代码/工具                                 |
| :---------------------------------- | :----------------------------------------------------------- | :--------------------------------------------------------- | :-------------------------------------------- |
| **1. 本地模型 (Ollama)**            | 在本地PC或服务器上拉取并运行模型，通过 **本地HTTP端口** 调用。 | 数据敏感、需要离线、快速原型验证本地模型能力。             | `OllamaLLM`, `init_chat_model("ollama/...")`  |
| **2. 本地模型 (vLLM/Transformers)** | 使用专业库高性能加载Hugging Face/ModelScope的模型权重文件。  | 生产环境部署、要求高吞吐量和低延迟。                       | `vLLM`, `HuggingFacePipeline`, **FastAPI**    |
| **3. 本地模型服务化 (FastAPI)**     | 将上述部署的模型包装成**标准HTTP API**（如OpenAI格式）对外提供服务。 | 团队共享、为多个应用提供统一模型服务、实现模型与应用解耦。 | `FastAPI`, `openai` 兼容接口                  |
| **4. 第三方API (OpenAI格式)**       | 调用遵循 **OpenAI API 规范** 的国内外模型服务。              | 快速开发、无需管理模型基础设施、使用最先进的闭源模型。     | `init_chat_model("openai/...")`, `ChatOpenAI` |

### 🚀 详解与调用示例

#### 1. 使用 `init_chat_model()` 统一调用

这是 LangChain 推荐的现代方式，用一个函数统一配置各种后端模型，非常清晰。

```python
from langchain.chat_models import init_chat_model

# 方式一：调用云端遵循 OpenAI 协议的模型（如 DeepSeek, Together.AI 等）
chat = init_chat_model(
    "openai/my-favorite-model", # 或 "openai/gpt-4o"
    base_url="https://api.deepseek.com/v1",
    api_key="your_deepseek_api_key",
    model="deepseek-chat" # 有些平台需要在此指定模型名
)

# 方式二：调用本地 Ollama 服务的模型
chat = init_chat_model(
    "ollama/qwen2.5:7b", # 格式: `ollama/<model_name>`
    base_url="http://localhost:11434/v1", # Ollama 的 OpenAI 兼容端点
    api_key="none" # 通常不需要 key，但必须提供（可用任意字符串）
)

# 统一调用方式
response = chat.invoke("你好！请做自我介绍。")
print(response.content)
```



#### 2. 本地模型部署与调用详解

本地部署的核心是 **“先服务化，后调用”**。即先让模型在一个进程内运行并监听一个端口，然后再通过HTTP客户端去访问这个端口。

**方案A：使用 Ollama (最简单)**

Ollama 帮你完成了所有服务化的工作，你只需要下载和运行。

1. **部署模型（在服务器或本地命令行）**:

   ```bash
   # 拉取并运行模型，自动在 localhost:11434 提供 API
   ollama pull qwen2.5:7b
   ollama run qwen2.5:7b # 这个命令是交互式的，后台运行可用：ollama serve
   ```

   

2. **在 LangChain 中调用**:

   ```python
   from langchain.chat_models import init_chat_model
   
   # 方法 1: 使用 init_chat_model (推荐)
   llm_ollama = init_chat_model(
       "ollama/qwen2.5:7b",
       base_url="http://localhost:11434/v1", # 关键！使用 /v1 端点
       api_key="none",
       temperature=0.7
   )
   
   # 方法 2: 使用 ChatOllama (较旧)
   from langchain_community.chat_models.ollama import ChatOllama
   llm_ollama = ChatOllama(model="qwen2.5:7b")
   
   # 调用
   response = llm_ollama.invoke("为什么天空是蓝色的？")
   print(response.content)
   ```

   

**方案B：使用 vLLM + FastAPI (高性能、生产级)**

这是更专业的部署方式，适合拥有GPU服务器的情况。

1. **部署模型服务（在GPU服务器上）**:

   ```bash
   # 1. 使用 vLLM 启动一个兼容 OpenAI API 的服务
   vllm serve huggingface/model-path --host 0.0.0.0 --port 8000
   
   # 2. 或者，编写一个自定义的 FastAPI 应用来封装模型，提供更灵活的 control
   # app.py
   from fastapi import FastAPI, Security, HTTPException
   from fastapi.security import APIKeyHeader
   from pydantic import BaseModel
   from transformers import pipeline
   import uvicorn
   
   app = FastAPI()
   api_key_header = APIKeyHeader(name="X-API-Key")
   VALID_API_KEY = "your_secret_key_here" # 设置密钥
   
   # 加载模型 (可根据需要替换为 vLLM 或其他推理引擎)
   pipe = pipeline("text-generation", model="your/local/model/path", device=0)
   
   class Request(BaseModel):
       prompt: str
       max_length: int = 100
   
   @app.post("/generate")
   async def generate_text(request: Request, api_key: str = Security(api_key_header)):
       if api_key != VALID_API_KEY:
           raise HTTPException(status_code=401, detail="Invalid API Key")
       result = pipe(request.prompt, max_length=request.max_length)
       return {"generated_text": result[0]['generated_text']}
   
   if __name__ == "__main__":
       uvicorn.run(app, host="0.0.0.0", port=8000)
   ```

   

   运行服务：`python app.py`

2. **在任何客户端调用自定义服务**:

   ```python
   import requests
   
   response = requests.post(
       "http://your-server-ip:8000/generate",
       json={"prompt": "法国的首都是", "max_length": 50},
       headers={"X-API-Key": "your_secret_key_here"} # 密钥认证
   )
   print(response.json())
   ```

   

3. **在 LangChain 中调用这个自定义服务**:

   ```python
   # 如果服务兼容 OpenAI 格式，可以直接用 init_chat_model
   chat = init_chat_model(
       "openai/any-model-name", # 模型名可任意，因为请求会被转发到你的服务
       base_url="http://your-server-ip:8000/v1", # 你的服务地址
       api_key="your_secret_key_here" # 你的密钥
   )
   
   # 如果服务是自定义格式，可以继承 BaseChatModel 编写一个自定义类，或者直接使用 requests。
   ```

   

#### 3. 调用 Hugging Face 模型文件

对于直接从 Hugging Face 或 ModelScope 下载的权重，通常使用 `Transformers` 库加载，然后集成到 LangChain。

```python
from langchain.chat_models import init_chat_model
from langchain.llms import HuggingFacePipeline
import transformers

# 1. 使用 Transformers pipeline 加载本地模型
pipe = transformers.pipeline(
    "text-generation",
    model="/path/to/your/downloaded/model", # 权重文件路径
    device="cuda:0", # 使用GPU
    model_kwargs={"torch_dtype": torch.float16} # 半精度加载节省显存
)

# 2. 包装成 LangChain 的 LLM
llm_local = HuggingFacePipeline(pipeline=pipe)

# 3. 直接调用
response = llm_local.invoke("请写一首关于秋天的诗。")
print(response)

# 注意：HuggingFacePipeline 返回的是字符串，不是 ChatMessage。
# 如果需要聊天格式，可以用 ChatPromptTemplate 处理好提示再传入。
```



### 💎 最终总结与建议

1. **首选 `init_chat_model()`**：这是 LangChain 现在更推荐的方式，接口统一，配置灵活，未来兼容性更好。
2. **本地部署的本质是“模型服务化”**：
   - **简单测试/个人用**：**Ollama** 是零配置的最佳选择，`init_chat_model("ollama/...")` 直接调用。
   - **生产环境/团队用**：采用 **vLLM** 或 **FastAPI** 部署为独立服务。这样做的好处是：
     - **解耦**：应用代码和模型环境分离，更新模型无需重启应用。
     - **资源共享**：一个模型服务可供多个项目或团队使用。
     - **负载均衡**：可以启动多个模型实例，用 Nginx 做负载均衡。
     - **安全管控**：通过 API 密钥、速率限制等手段控制访问。
3. **调用链路的两种模式**：
   - **内嵌模式**：模型与应用在同一进程，如 `HuggingFacePipeline`。**缺点**：模型加载慢、易阻塞主进程、资源独占性强。
   - **服务模式**：模型作为独立服务，应用通过 HTTP/gRPC 调用。**优点**：稳定、高效、易扩展，是生产环境的**标准做法**。你提到的“服务器部署，接口访问”正是这种模式的完美实践。
